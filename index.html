Master Claude Prompt

Copy this prompt at the start of every new Claude session. It sets context so every explanation is tutor-quality and exam-focused.

// MASTER SESSION PROMPT — PASTE AT START OF EVERY CHAT
You are my dedicated SDE 2 interview coach. I have a CSE background and am preparing for Google, Microsoft, NVIDIA, and similar top-tier companies. I am following the Striver A–Z DSA Sheet from scratch (zero to advanced), and I also want to cover System Design, OS, DBMS, Computer Networks, and OOP. For every topic or problem I bring to you: 1. Explain the CONCEPT first with a real-world analogy 2. Walk through the BRUTE FORCE → OPTIMIZED approach step by step 3. Show clean, interview-ready CODE in Java (or C++ if I ask) 4. Give TIME and SPACE complexity analysis 5. List 3–5 SIMILAR PROBLEMS I should solve next 6. Give me a SHORT "interview cheat line" — one sentence to explain my approach to an interviewer 7. End with a GITHUB NOTE block I can paste directly into my repo's markdown file My GitHub repo structure is: dsa-journey/ with folders per topic (arrays/, linked-list/, trees/, etc.). Each folder has a README.md with problem notes and a solutions/ subfolder. Tone: like a senior engineer mentoring a junior — honest, no fluff, interview-focused. Flag if a problem is "Must-Know" for FAANG.
// TOPIC DEEP-DIVE PROMPT
I am starting the [TOPIC NAME] section of Striver A–Z sheet. Topic: [e.g., Dynamic Programming — 1D DP] Please give me: 1. A CONCEPT MAP — all sub-patterns in this topic (numbered list) 2. The MOST IMPORTANT 5 problems for Google/Microsoft interviews from this topic 3. The KEY INSIGHT or "aha moment" that unlocks this topic 4. Common MISTAKES candidates make in interviews for this topic 5. A REVISION CHECKLIST I can add to my GitHub README for this topic After this, I'll give you problems one by one — be ready to tutor me.
// DAILY PRACTICE PROMPT
Today's DSA session — [DATE] Current Striver Sheet position: [Step X.Y — Topic Name] I solved / attempted this problem: [PROBLEM NAME] My approach: [describe your thinking or paste your code] Please: 1. Review my approach and code — give honest feedback 2. If wrong/suboptimal, show me the optimal solution with explanation 3. Give me the GitHub note block for today's commit 4. Rate this problem: Easy / Medium / Hard for Google interviews 5. Should I revisit this in 3 days? (spaced repetition flag)
// SYSTEM DESIGN SESSION PROMPT
I am preparing System Design for SDE 2 interviews at Google, Microsoft, and NVIDIA. Today's topic: [e.g., Design a URL Shortener / Design Netflix / CAP Theorem] Please structure your answer as: 1. CLARIFY requirements (functional + non-functional) — show me how to ask the right questions 2. HIGH-LEVEL DESIGN — components, diagram description 3. DEEP DIVE — the most important component (DB schema, API design, etc.) 4. SCALE — how to handle 10x, 100x traffic 5. TRADE-OFFS — what did we sacrifice and why 6. NVIDIA-SPECIFIC angle — where does GPU/parallel computing fit if relevant Also give me a GitHub markdown template I can store in my system-design/ folder.

1

Create & Initialize the Repo

Go to GitHub → New Repository → name it dsa-journey. Add a description: "Striver A–Z + System Design prep for FAANG SDE 2". Initialize with README. Clone locally: git clone https://github.com/YOUR_USER/dsa-journey

2

Create Folder Structure (one time)

Run: mkdir arrays linked-list trees graphs dp stack-queue binary-search heaps tries greedy system-design revision-notes — then add a blank README.md in each. Push this skeleton as your first commit: git add . && git commit -m "init: repo skeleton" && git push

3

Daily Commit Workflow

After each problem: create a .java file in the correct folder → add Claude's note block to the topic README → commit with a meaningful message like: feat(arrays): Two Sum - HashMap O(n) approach → push. This builds your green contribution graph and interview evidence.

4

GitHub Issues as a Revision Tracker

For every problem you need to revisit: create a GitHub Issue titled "Revisit: [Problem Name]" with labels revisit-3d, revisit-7d, or must-master. Close the issue when you solve it confidently. This is free spaced repetition tracking.

5

GitHub Projects Board (Kanban)

Create a Project board with columns: To Do → In Progress → Solved Once → Confident. Add all Striver sheet problems as cards. Move them as you progress — this is your visual progress tracker at a glance.

6

Weekly Review Commit (Sunday ritual)

Every Sunday: update revision-notes/weekly-review.md — what you solved, what patterns clicked, what you're weak at. Commit this as chore: week-N review. Over months, this becomes a powerful personal interview prep diary.

Color-Coded Notes System

Use this system in your physical notebook, Notion, or Obsidian. Every problem note follows the same format and color.

// PROBLEM NOTE TEMPLATE
PROBLEMProblem Name + LeetCode number
DATEDD-MM-YYYY
DIFFICULTY🔴 Hard / 🟡 Med / 🟢 Easy
PATTERNTwo Pointer / Sliding Window / etc.
INTUITIONWhy does this approach work?
BRUTENaive approach + complexity
OPTIMALBest approach + complexity
INTERVIEW"I will use X because Y in O(Z) time"
REVISITDate to review again

Highlighter colors for physical notebook or Notion callout blocks:

🟩 GREEN — KEY INSIGHT
The one realization that solves the problem. Write it in big bold text.
🟨 YELLOW — PATTERN NAME
Sliding Window, DP on Trees, Dijkstra variant, etc.
🔴 RED — EDGE CASES
Empty array, single element, overflow, negative numbers. These lose you interviews.
🔵 BLUE — CODE TEMPLATE
The reusable code skeleton for this pattern. Copy-paste start for new problems.
🟣 PURPLE — COMPLEXITY
Time + Space. Always write this — it's the first thing interviewers ask.
🟠 ORANGE — INTERVIEW LINE
One sentence verbal explanation. Practice saying this out loud.

System Design Checklist

Concepts to Master

  • CAP Theorem + trade-offs
  • Horizontal vs Vertical scaling
  • Load balancers (L4 vs L7)
  • CDN and caching strategies
  • SQL vs NoSQL — when to use
  • Message queues (Kafka, RabbitMQ)
  • Consistent hashing
  • Rate limiting algorithms

Must-Design Systems

  • URL Shortener (start here)
  • Twitter/X Feed
  • Netflix / YouTube
  • Uber / Ride-sharing
  • WhatsApp Messaging
  • Google Drive / Dropbox
  • Search Autocomplete
  • Distributed Cache (Redis)

Interview Framework

  • 1. Clarify requirements (5 min)
  • 2. Estimate scale (back of envelope)
  • 3. High-level diagram (10 min)
  • 4. Deep dive 1–2 components (15 min)
  • 5. Discuss trade-offs (5 min)
  • 6. Bottlenecks + improvements
